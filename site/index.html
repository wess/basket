Functional building blocks for native desktop apps.

What atlas is to web apps, basket is to desktop. Composable, functional Bun/TypeScript packages that snap together — typed IPC, windows, menus, sqlite, auth, AI — on a native webview. Tiny binaries, one language, no Rust.

Why basket

The pieces of a desktop app, done functionally.

Use one package or all 29. They're small, they're typed, and they snap together.

🪶

Tiny binaries

~3-8 MB, not ~150 MB. A native webview, not a bundled Chromium runtime.

🦋

One language

TypeScript on the host (Bun) and in the webview. No Rust, no Go — just one toolchain.

🧩

Composable

29 small packages. Use one, use them all — they're designed to snap together cleanly.

📡

End-to-end typed IPC

Typed channels, events, pipelines, validation, and structured errors shared host-to-webview.

🗄️

Data layer included

A bun:sqlite query builder, migrations, sandboxed FS, TTL cache, and a rotating logger.

🔐

Auth & secrets

Argon2id + OAuth (PKCE), plus the OS keychain on macOS, libsecret, and Credential Manager.

🤖

AI-friendly

Chat, streaming, and embeddings across OpenAI, Anthropic, and Ollama — plus MCP for app tools.

λ

Functional only

No classes. Immutable inputs, composition over inheritance — the same conventions as atlas.

📦

No npm tax

Vendor it as a workspace, edit in place, no version skew. Each package ships a concise AGENTS.md.

Packages

Each one wraps a butter primitive or a Bun-native API — nothing reinvented.

Quick start

A native window in three commands.

Scaffold a project with the CLI, install, and run. ⌘T toggles the theme; ⌘Q quits.

terminal 
# scaffold a new app (templates: minimal, menubar)
bunx --bun @basket/cli init myapp --template minimal
cd myapp

# install + run — a native window opens
bun install
bun run dev

Or vendor basket into an existing project as a workspace — it isn't on npm.

terminal 
# vendor as a workspace, then edit in place
curl -sL https://github.com/wess/basket/archive/refs/heads/main.zip -o /tmp/basket.zip
unzip -q /tmp/basket.zip -d /tmp/basket-expand
mv /tmp/basket-expand/basket-main ./basket
rm -rf /tmp/basket.zip /tmp/basket-expand
echo "basket/" >> .gitignore

# then add to package.json: "workspaces": ["basket/packages/*"]
# and "@basket/ipc": "workspace:*", etc.
